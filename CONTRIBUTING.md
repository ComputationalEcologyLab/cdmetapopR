# Contributing to cdmetapopR

Thank you for considering contributing to cdmetapopR!

## What kinds of contributions are we looking for?

We welcome these contributions:
- Bug reports
- Feature requests
We welcome also any suggestions for improvement on:
- Functionality 
- Documentation 
- Test coverage 

## Your First Contribution

New to contributing to open source? Check out [this guide](https://opensource.guide/how-to-contribute/) for helpful tips on getting started.


## How to Report a Bug

Before creating bug reports, please check the existing issues to avoid duplicates. When creating a bug report, include:
- A clear and descriptive title
- Steps to reproduce the issue
- Expected behavior vs. actual behavior
- Your R version and package version
- A minimal reproducible example if possible

## How to Suggest a Feature or Enhancement

Feature suggestions are welcome! Please:
- Check existing issues first
- Provide a clear description of the proposed feature
- Explain why this feature would be useful
